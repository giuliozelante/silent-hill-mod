Re: Question: weapon file clarification
Alright, forgive me for skimming through your post, but here's an explanation of the weapon kf2s, etc.

weapon_lod0/1/2.kf2 = The weapon mesh, in various levels of detail. Basically the weapon will switch to one of the meshes, depending on how near the camera is. Check the weapon's script for mesh LOD declaration, and distance defintions.


weapon_levelitem.kf2 = The mesh used for the level item -- i.e the weapon that is dropped on the floor. Doesn't use an animation, and the mesh is usually slightly larger than the 'actual' weapon mesh.


weapon_default.kf2= Animation used for when weapon is in default state -- i.e not being used.


weapon_shoot.kf2= Shooting animation, nothing special about it. Generally used to attach muzzel flare etc to on certain frames.


weapon_reload.kf2= Pretty obvious; don't think it's used that much, except for the sawed shot gun where the chamber opens up.


Weapon_weapontoprojectile.kfs = Not actually checked this one out; you're guess sounds pretty accurate though.


weapon_ricochet.kf2 = Not sure, but it's not actually used in the game so...

In fact, quite a few of these animations aren't even used (check the scripts) so I wouldn't worry about them. The main ones are default and shoot -- reload too if you're making a weapon that needs to do something fancy for reloading.

Re: Question: weapon file clarification
Not related to your current problem, but I think I've figured out what the two unknown animations do:

(W = any weapon, like deserteagle)

Both W_ricochet.kf2 and W_WeaponToProjectile.kf2 seem only to be used by enemy_W.txt (in the projectiles directory) for when an enemy drops that weapon. Specifically,

W_ricochet.kf2 = a looped animation played while the dropped weapon bounces. Usually (I think), it's 6 frames long and there isn't any real animation.

W_WeaponToProjectile.kf2 = an animation played while dropped weapon is actually falling. I'm pretty sure it doesn't loop, but if you have a custom ricochet animation that actually moves the bouncing weapon (probably not a good idea, though) this animation might start over. Usually (I think), this is 11 frames and most only use rotation.

I suppose you'd use WeaponToProjectile to rotate the weapon to match your level item's orientation, so there won't appear to be a big jump in animation, but I'm not sure when Ricochet would be needed.
