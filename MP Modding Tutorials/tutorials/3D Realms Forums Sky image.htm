It's possible to create own worldspheres. I created my own sky for a level of mine. It's a blue sky with white clouds. My worldsphere is called "bluesky". Here's what you have to do..:

1. Define a new sky in "worldsphere.txt" script in "data/database/worldspheres". Here's an example:
quote:
[bluesky] // Daedalus custom sky
{
[Geometry]
ExportData = bg_bluesky.kf2;
}
2. Use 3dsmax 3 or Milkshape and create a new "bg_mysky.kf2" file which includes the name of your TGA sky image.

3. Put everything in the folder "worldspheres" and your sky image in the sub-folder "textures".

Then you can use your new worldsphere in your level. Like Rama said you put the name of it in the levels.txt script (in my case "bluesky").
That's it.
p.s. you can make smaller skies than 1024x256. Mine is 256x256 only and it works too. You have to create a new .kf2 with Milkshape and set up everything there.

btw. if you don't want to create new .kf2 files then use the existing ones and just replace the sky image itself. But then you have to use the same filename and dimension, e.g. 1024x256 or you will experience strange gfx errors in the game.

In your case it's better to use the existing bg_sunset.kf2 which refers to bg_sunset_all.tga image. So you just replace the TGA. But note that if you replace the image only, not create a new worldsphere actually, then the original sunset can't be used anymore. The advantage of what I did (creating a new worldsphere) is that I can use all skies, because I didn't replace anything.
