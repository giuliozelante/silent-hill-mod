Re: Weapons Animation
Erm, I dunno about hex editing, but weapon/kf2 animation is pretty straightforward (for mp1 at least). For simplicity and to avoid conflicts, it's best to animate on the same file as the model you're using (it's complicated if you use several different LOD models, so I wouldn't bother for your first one). Btw, I'd advise against trying to replace/animate the beretta weapon until you've done this correctly a few times. The beretta is one of the few weapons in the game that uses 2-3 different LOD meshes, and each mesh has a different set of joint names -- get this wrong and the game will often crash. My advice would be to start with a brand new weapon.

Normally the game will recognise any mesh you export from 3ds max as a seperate joint, so you don't actually need to set up complicated bone systems or anything. You may, however, want to group mesh/objects together by using the link button (it looks like two boxes chained together by two small squares). Once you've got it set up, all you need to do is turn on the 'animate' button, animate your weapon how you like, and then export. (NB, as far as I know, you can't 'rig' weapon models, so the mesh might tear if you try very elaborate animations)

One useful trick I learnt is that if you hide an animated mesh, the geometry won't export, but the animation data will. What you can then do is link a child mesh to this, and have much more control over where the mesh pivots (there's probably a MUCH better way to do this, but it works for me). Very useful when you're rotating objects, because often the pivot point isn't set in the centre of mass.

Exporting should be the same as for regular character animations -- tick export 'animation' and 'remove hidden', plus any additional options you want (such as looping). You'll need to export the model, a 'default' (non-moving) animation, and any additional animations. Provided you use the same mesh for all files (i.e. the joints are the same), you won't need to export the mesh again with the animation files (what I do is export the default animation along with the model into the same kf2 file, then additional animations seperately). Then it's just a case of referring to the correct kf2 file in the weapon's text file.

Incidentally, this should work for almost any kf2 object that supports animation, such as projectiles, worldspheres, and skin items. You simply add an additional block called [animationset] and then [animation] blocks within (refer to weapon & projectiles files for reference). You can create some very cool effects with animated projectiles (plus they have simple physics). Both the decapitated heads and sliced off torsos in Katana were animated projectiles.

Try experimenting, and see what you can animate!
