Maddieman

Re: Enemy bullet dodging
    06/23/03 05:38 AM

quote:
it says that the receiver cannot handle such messages. But I thought that the this class points to the enemy it has hit, right?
Wrong. "This" points to the projectile...

i.e. If the projectile hits a character, then do this to the projectile (e.g. create a new projectile, or start a particle effect).
quote:
I also was wondering how you can send messages from a projectile to the thing it hit, can someone tell me?
Unfortunately, this can't be done, which makes life just that little bit more difficult. The "Player" reciever isn't recognised, either.

So put simply, the only way to get your enemies to respond to player input is through gethit and death animations. Thankfully, this is more flexible than you might think.

About animation switching -- it tends to interrupt the current animation (look at wallrunning for example). The problem with using idle animations is that they restart if the player moves -- even if it's just looking around; so make sure you disable the player controls durning the animation.

Check out this thread for further information on projectiles, etc:

http://forums.3drealms.com/ubb/ultimatebb.php?ubb=get_topic;f=18;t=001880
